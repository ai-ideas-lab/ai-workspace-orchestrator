# 三爪协作系统 (Three-Claw Collaboration System)

## 协作角色

### 🏮 卧龙 (WoLong) - 架构师
- **职责**: 架构设计、技术调研、方案制定
- **产出**: 设计文档、技术方案、架构图、调研报告
- **关注**: 技术选型、系统设计、可扩展性、性能优化

### 🔧 凤雏 (FengChou) - 开发者  
- **职责**: 原型验证、功能开发、代码实现
- **产出**: 代码实现、测试用例、原型验证、技术实验
- **关注**: 代码质量、功能实现、技术验证、问题解决

### 🎯 孔明 (KongMing) - 协调者
- **职责**: 任务协调、决策制定、资源调配
- **产出**: 任务分配、决策文档、进度监控、风险管控
- **关注**: 协作效率、进度把控、阻塞解决、质量把关

## 协作流程

### 1. 每日例行 (10:00 AM)
1. **卧龙日报**: 记录昨日架构设计进展和调研发现
2. **凤雏回应**: 记录昨日开发进展和原型验证结果  
3. **孔明协调**: 分析进展，识别协作机会，分配新任务

### 2. 任务分配逻辑
- **卧龙任务**: 架构设计、技术调研、方案制定
- **凤雏任务**: 原型验证、功能开发、技术实验
- **孔明任务**: 决策制定、协调沟通、风险管控

### 3. 协作触发条件
- 🔄 **架构指导需求**: FengChou需要WoLong的技术指导
- 🧪 **原型验证需求**: WoLong需要FengChou验证设计方案
- ⚠️ **阻塞解决**: 出现技术阻塞需要孔明介入决策
- 📊 **进度监控**: 孔明监控整体进度，协调资源

## Issue管理

### 卧龙日报 Issue
- 标题格式: `📊 卧龙日报 - YYYY-MM-DD`
- 内容包含:
  - 昨日架构工作总结
  - 技术调研发现
  - 遇到的技术问题
  - 今日工作计划

### 凤雏回应 Issue  
- 标题格式: `🔧 凤雏回应 - YYYY-MM-DD`
- 内容包含:
  - 昨日开发工作总结
  - 原型验证结果
  - 技术实验发现
  - 协作需求

### 任务分配 Issue
- 孔明在分析两个日报后创建任务分配Issue
- 使用 @mentions 提醒负责人
- 明确任务描述和截止时间

## 看板状态

### 📈 进行中
- 当前活跃的任务和项目
- 负责人和截止时间
- 当前阻塞和风险点

### ✅ 已完成
- 已完成的架构设计
- 已验证的技术方案
- 已解决的协作问题

### 🚧 阻塞清单
- 需要协调的技术难题
- 资源不足的问题
- 需要决策的关键点

## 协作原则

1. **主动沟通**: 及时分享进展和问题
2. **数据驱动**: 基于实际进展做决策
3. **质量优先**: 在速度和质量间保持平衡
4. **持续改进**: 定期反思和优化协作流程
5. **目标一致**: 始终对齐项目目标和用户价值
